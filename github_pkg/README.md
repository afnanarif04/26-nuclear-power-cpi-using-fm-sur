# Fully Modified SUR–ARDL Analysis with EKC Application

Replication code and data for a panel cointegrating polynomial regression study
estimating the long-run relationship between a sector-specific share variable and a
consumer price index across a balanced panel of eight economies (1995–2021).

The empirical strategy layers four estimators under different assumptions about
cross-sectional dependence, slope heterogeneity, and integration order:

1. **FM-SUR** — System fully modified seemingly unrelated regression (Wagner, Grabarczyk & Hong, 2020)
2. **Group-mean FM-OLS** — Cross-section dependence robust group-mean estimator (Wagner & Reichold, 2023)
3. **Pooled FM-OLS** — Pooled fully modified estimator (de Jong & Wagner, 2022)
4. **Bounds-test ARDL** — Conditional error-correction model accommodating mixed I(0)/I(1) regressors (Pesaran, Shin & Smith, 2001)

Panel causality is assessed using the Dumitrescu–Hurlin (2012) heterogeneous non-causality test.

## Requirements

- R ≥ 4.2.0
- RStudio (recommended, for `rstudioapi::getActiveDocumentContext()`)

Install all required packages in one command:

```r
install.packages(c(
  "readxl",    # read Excel data file
  "dplyr",     # data manipulation
  "tidyr",     # reshaping
  "plm",       # panel data models, CD test
  "Matrix",    # sparse matrix operations (FM-SUR)
  "MASS",      # generalised inverse (fallback)
  "sandwich",  # heteroskedasticity-robust covariance
  "car",       # linear hypothesis tests (bounds-test)
  "urca",      # unit root and cointegration tests
  "ggplot2"    # descriptive plots
))
```

## How to run

**Option A — run everything at once (recommended)**

1. Open `R/00_RUN_ALL.R` in RStudio.
2. Click **Source** (Ctrl + Shift + S on Windows/Linux; Cmd + Shift + S on macOS).
3. All ten scripts execute in order. CSV output tables appear in `output/`.

**Option B — run scripts individually**

Open any script in RStudio and click **Source**. Each script calls
`setwd(dirname(rstudioapi::getActiveDocumentContext()$path))` at the top, so it
automatically sets the working directory to the `R/` folder. The scripts read from
`../data/` and write to `../output/` relative to `R/`.

> **Note.** If you are not using RStudio, replace the `setwd(...)` line in each
> script with `setwd("/path/to/your/R/folder")` before sourcing.
